# 全局样式基础规范计划

## 一、产品性质定性

### 1.1 产品定位
**面试虎 - AI智能面试助手**
- 核心功能：实时语音识别、双知识库RAG检索、智能问答生成
- 目标用户：技术岗位求职者、面试准备者
- 使用场景：面试现场实时辅助、面试练习模拟

### 1.2 设计调性
| 维度 | 定性描述 |
|------|----------|
| **科技感** | 紫色/蓝色渐变、发光效果、玻璃拟态 |
| **专业感** | 清晰层级、规范间距、可读字体 |
| **智能感** | 动态渐变、流畅动画、状态反馈 |
| **信任感** | 稳定配色、明确交互、安全提示 |

### 1.3 参照SVG主题提取
- **主色调**：金色渐变（老虎品牌色）#fbbf24 → #f59e0b → #d97706
- **科技色**：紫色渐变（AI智能色）#7C3AED → #A78BFA → #C084FC
- **辅助色**：蓝色（专业冷静）#60a5fa、粉色（活力）#f472b6
- **背景色**：深色系（沉浸体验）#0F0F23 → #1A1A3E
- **效果**：发光(glow)、模糊(blur)、浮动(float)

## 二、样式规范架构

### 2.1 设计系统三层架构
```
┌─────────────────────────────────────────┐
│           组件层 (Component)            │
│  Button / Card / Input / Badge / ...   │
├─────────────────────────────────────────┤
│           语义层 (Semantic)             │
│  Primary / Secondary / Success / ...   │
├─────────────────────────────────────────┤
│           基础层 (Primitive)            │
│  Color / Typography / Spacing / ...    │
└─────────────────────────────────────────┘
```

### 2.2 文件结构规划
```
frontend/src/
├── assets/
│   └── main.css              # 全局样式入口
├── styles/
│   ├── variables.css         # CSS变量定义
│   ├── components.css        # 组件样式类
│   ├── utilities.css         # 工具类样式
│   └── animations.css        # 动画定义
└── tailwind.config.js        # Tailwind配置
```

## 三、具体实现步骤

### 步骤1：Tailwind配置升级
- 扩展颜色系统（基础色、语义色、功能色）
- 扩展字体系统（标题、正文、代码）
- 扩展动画系统（发光、浮动、渐变）
- 扩展阴影系统（玻璃拟态、发光阴影）
- 扩展间距系统（8px基准）

### 步骤2：CSS变量定义
- 定义设计令牌（design tokens）
- 定义颜色变量（--color-primary, --color-secondary...）
- 定义间距变量（--spacing-xs, --spacing-sm...）
- 定义圆角变量（--radius-sm, --radius-md...）
- 定义阴影变量（--shadow-glass, --shadow-glow...）

### 步骤3：组件样式规范
| 组件 | 样式规范 |
|------|----------|
| **Button** | 主按钮（渐变）、次按钮（玻璃）、轮廓按钮、危险按钮 |
| **Card** | 基础卡片、增强卡片、功能卡片 |
| **Input** | 玻璃输入框、聚焦状态、错误状态 |
| **Badge** | 状态徽章、标签徽章 |
| **Dialogue** | 用户气泡、AI气泡、问题标签、答案区域 |
| **Modal** | 玻璃模态框、动画入场 |

### 步骤4：动画效果规范
- `float-slow` - 慢速浮动（Logo）
- `float-medium` - 中速浮动（装饰元素）
- `pulse-glow` - 脉冲发光（状态提示）
- `gradient-shift` - 渐变流动（按钮）
- `fade-in-up` - 淡入上浮（页面入场）
- `slide-in-right` - 右侧滑入（弹窗）

### 步骤5：响应式规范
- 移动端（< 640px）：紧凑布局、大触控区
- 平板端（640px - 1024px）：中等布局
- 桌面端（> 1024px）：完整布局、更多内容

## 七、风险与注意事项

### 7.1 兼容性风险
- 玻璃拟态效果在旧浏览器可能不支持
- SVG动画在部分移动端浏览器性能较差
- 渐变动画在低端设备可能卡顿

### 7.2 性能风险
- 过多模糊效果影响性能
- 复杂动画在低端设备卡顿

### 7.3 解决策略
- 使用 CSS `backdrop-filter: blur()` 降级方案
- 移动端减少动画复杂度
- 使用 `will-change` 优化动画性能

### 8.1 视觉一致性

- 所有组件使用统一的间距规范
- 所有交互使用统一的动画效果

### 8.2 响应式验证
- 移动端布局正常（< 640px）
- 平板端布局正常（640px - 1024px）
- 桌面端布局正常（> 1024px）

### 8.3 交互验证
- 按钮hover/active状态正常
- 输入框focus状态正常
- 弹窗动画流畅
