#### 能力图标设计

### 2.3 色彩方案

延续现有科技渐变风格，扩展为更丰富的能力色彩体系：

```
主色调：深蓝 (#1e1b4b) → 紫色 (#4c1d95)
能力色：
  语音识别 → 天蓝 (#3b82f6)
  知识库   → 紫罗兰 (#8b5cf6)
  交互对话 → 粉紫 (#d946ef)
  RAG检索  → 青色 (#06b6d4)
  AI智能   → 金色 (#f59e0b)
高光色：白色渐变 + 发光效果
```

### 2.4 设计风格要求

1. **简洁而不简单**：整体简洁，但细节丰富，体现技术深度
2. **动态感**：通过渐变、发光、波纹等效果营造动态交互感
3. **科技感**：现代扁平化风格，符合 AI 产品气质
4. **可扩展性**：Logo 可单独使用，也可配合文字使用

### 3.2 SVG 技术实现要点

#### 3.2.1 Logo 主体 (logo.svg)

**尺寸**：512 x 512 px

**图层结构**：

| 图层 | 内容 | 技术实现 |
|------|------|----------|
| 背景层 | 圆形渐变背景 | radialGradient |
| 虎首层 | 抽象虎首轮廓 | path 路径 + 圆角 |
| 声波层 | 语音波纹效果 | path 波浪线 + 动画 |
| 知识网络层 | 知识节点与连接 | circle + line + 渐变 |
| 能力图标层 | 5个能力图标 | 独立分组 + 发光效果 |
| 文字层 | "面试虎" + "Interview Tiger" | text + filter |

**关键 SVG 特性**：
- 使用 `<defs>` 定义渐变、滤镜、动画
- 使用 `<filter>` 实现发光效果 (feGaussianBlur + feMerge)
- 使用 `<animate>` 实现声波动态效果
- 使用 `<g>` 分组管理各模块

#### 3.2.2 横幅更新 (hero.svg)

**更新内容**：

- 增强背景动态效果（增加粒子动画）

## 四、潜在风险与应对

### 4.1 风险清单

| 风险 | 描述 | 应对策略 |
|------|------|----------|
| 设计复杂度 | 图标过多可能导致视觉混乱 | 控制图标数量，保持留白 |
| 动画性能 | SVG 动画可能影响页面加载 | 使用 CSS 动画替代部分 SVG 动画 |
| 跨平台兼容性 | 部分 SVG 特性在旧浏览器不支持 | 使用基础 SVG 特性，避免高级滤镜 |
| 视觉一致性 | 新设计与现有风格冲突 | 严格遵循现有色彩方案和设计语言 |

### 4.2 验证标准

1. **视觉效果**：Logo 在不同背景下清晰可见
2. **尺寸适应性**：缩小到 64x64 仍能辨认
3. **动画流畅度**：声波动画流畅无卡顿
4. **风格一致性**：与现有 hero.svg 风格统一
