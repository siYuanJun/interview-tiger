# 方案设计文档

## 项目信息
- 项目名称: interview-tiger
- 需求描述: 修复语音识别延迟（添加手动结束按钮）+ UI整体翻新
- 创建时间: 2026-07-02
- 状态: 已完成

---

## R1: 语音识别延迟问题 - 方案A

### 方案描述
在语音识别组件中添加手动"完成"按钮，点击后立即提交当前识别文本。

### 技术实现
1. 在 `useSpeech.ts` 中新增 `forceStop()` 方法
   - 调用 `recognition.stop()` 强制停止识别
   - 将当前累积的识别文本作为最终结果提交

2. 在 `InterviewPage.vue` 中添加完成按钮
   - 按钮显示时机: 正在识别时
   - 按钮文案: "完成"
   - 点击事件: 调用 `forceStop()`

### 优缺点
| 维度 | 优点 | 缺点 |
|------|------|------|
| 用户体验 | 响应迅速，用户可自主控制 | 增加一个交互按钮 |
| 技术复杂度 | 实现简单，改动小 | 无 |
| 风险 | 低 | 无 |

## R2: UI整体翻新 - 方案A

### 方案描述
采用 Glassmorphism 风格进行整体 UI 翻新。

### 技术实现
1. **颜色系统**
   - 主色: Navy (#1E3A5F)
   - 辅助色: Green (#2ECC71)
   - 背景: 渐变深蓝

2. **玻璃效果实现**
   ```css
   .glass-card {
     background: rgba(255, 255, 255, 0.1);
     backdrop-filter: blur(10px);
     border: 1px solid rgba(255, 255, 255, 0.2);
   }
   ```

3. **字体配置**
   - Poppins: 标题字体
   - Open Sans: 正文字体

### 实施范围
- `main.css`: 全局样式变量和基础样式
- `InterviewPage.vue`: 主界面样式
- `DialogueItem.vue`: 对话卡片样式
- `App.vue`: 布局和容器样式

### 优缺点
| 维度 | 优点 | 缺点 |
|------|------|------|
| 视觉效果 | 现代感强，科技感十足 | 需要适配不同浏览器 |
| 性能 | CSS实现，性能优秀 | backdrop-filter在部分旧浏览器不支持 |
| 风险 | 中 | 需测试浏览器兼容性 |
