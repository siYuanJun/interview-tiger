# 大模型调用模块 - 模块摘要

## 📌 一句话概括

实现火山引擎方舟大模型的调用服务，支持非流式和流式两种模式，处理API认证、请求构建、响应解析和错误处理。

---

## 👤 面向产品/业务人员

### 这个模块做了什么？

大模型调用模块是面试虎的"大脑思考能力"，负责根据面试问题和知识库内容生成个性化的回答建议。当系统识别到面试官的问题后，会将问题发送给火山引擎的大模型，大模型会结合用户的简历信息（如果配置了知识库），生成专业的回答建议。用户可以看到实时的打字效果，就像有人在帮你思考回答一样。

### 用户可以做什么？

| 用户故事 | 操作路径 | 完成状态 |
|---------|---------|----------|
| 获取AI生成的回答 | 提出问题后自动生成 | ✅ |
| 实时看到回答生成过程 | 打字机效果逐字显示 | ✅ |
| 调整回答风格 | 通过配置温度参数控制 | ✅ |
| 限制回答长度 | 通过配置最大token数控制 | ✅ |

### 涉及几个页面？

此模块为后端服务，通过API调用，前端面试页面展示结果。

## 🔧 面向技术负责人

### 涉及接口清单

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | /api/generate | 非流式大模型调用 |
| POST | /api/generate/stream | 流式大模型调用（SSE） |
| POST | /api/question | 完整问答流程（非流式） |
| POST | /api/question/stream | 完整问答流程（流式） |

### 关键数据结构

```
LLMService (服务类)
├── call_llm(messages, api_key, **kwargs)      # 非流式调用
├── call_llm_stream(messages, api_key, **kwargs) # 流式调用
├── _build_headers(api_key)                    # 构建请求头
└── _build_request_body(messages, **kwargs)    # 构建请求体

ChatMessage
├── role: "user" | "assistant" | "system"
└── content: string
```

### 关键架构决策

| ADR编号 | 决策内容 | 影响面 |
|---------|---------|--------|
| ADR-007 | 选择火山引擎方舟平台作为LLM服务 | 大模型调用模块 |
| ADR-008 | 使用SSE实现流式响应 | 实时展示模块 |

### 风险与注意事项

| 风险项 | 级别 | 说明 | 缓解措施 |
|--------|------|------|----------|
| API调用成本 | 🟡中 | 大模型调用有费用 | 建议添加使用量监控和预算控制 |
| 响应延迟 | 🟡中 | 大模型响应可能较慢 | 使用流式模式减少感知延迟 |
