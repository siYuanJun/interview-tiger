# 面试对话展示模块 - 模块摘要

## 📌 一句话概括

实现面试对话的展示和交互，支持左右分栏布局（PC端）、上下布局（移动端）、流式打字机效果、状态指示器和对话记录导出。

---

## 👤 面向产品/业务人员

### 这个模块做了什么？

面试对话展示模块是面试虎的"脸面"，负责展示整个面试过程。用户进入面试页面后，会看到一个清晰的界面，左侧（或上方）显示面试官的问题，右侧（或下方）显示AI生成的回答。回答会像打字机一样逐字显示，让用户有更好的体验。同时，界面会显示当前状态（录音中、识别中、生成中），让用户知道系统在做什么。面试结束后，用户可以导出对话记录。

### 用户可以做什么？

| 用户故事 | 操作路径 | 完成状态 |
|---------|---------|----------|
| 查看问题和回答 | 面试页面自动展示 | ✅ |
| 实时看到回答生成 | 打字机效果逐字显示 | ✅ |
| 了解系统状态 | 状态指示器显示当前状态 | ✅ |
| 导出对话记录 | 点击导出按钮 | ✅ |
| 在手机上使用 | 自动切换为上下布局 | ✅ |

### 涉及几个页面？

| 页面名称 | 功能 | 状态 |
|---------|------|------|
| 面试页面 | 对话展示、状态指示、导出功能 | ✅ |

| POST | /api/question/stream | 流式问答接口 |

### 关键数据结构

```
Dialogue (对话对象)
├── id: string                # 唯一标识
├── type: "question" | "answer"
├── content: string           # 内容
├── timestamp: number         # 时间戳
├── status: "pending" | "completed" | "error"
└── knowledgeUsed: boolean    # 是否使用知识库

InterviewStore (状态管理)
├── dialogues: Dialogue[]     # 对话列表
├── isRecording: boolean      # 是否正在录音
├── isGenerating: boolean     # 是否正在生成
├── currentQuestion: string   # 当前问题
├── addQuestion(text)         # 添加问题
├── addAnswer(answer)         # 添加回答
└── exportDialogues(format)   # 导出对话
```

### 关键架构决策

| ADR编号 | 决策内容 | 影响面 |
|---------|---------|--------|
| ADR-010 | 响应式布局：PC左右分栏+移动端上下布局 | 对话展示模块 |

### 风险与注意事项

| 风险项 | 级别 | 说明 | 缓解措施 |
|--------|------|------|----------|
| 大量对话卡顿 | 🟡中 | 对话过多时渲染性能下降 | 实现虚拟滚动或分页加载 |
| 内存泄漏 | 🟡中 | SSE连接未正确关闭 | 添加组件卸载时的清理逻辑 |
