# 录音与语音识别模块 - 模块摘要

## 📌 一句话概括

实现浏览器端实时录音和语音转文字功能，支持麦克风权限请求、音频采集、Web Speech API识别、问题判断和重复过滤。

---

## 👤 面向产品/业务人员

### 这个模块做了什么？

录音与语音识别模块是面试虎的"耳朵"，负责将面试官说的话转换成文字。用户点击开始面试后，系统会请求麦克风权限，然后实时监听面试官的语音，自动转换成文字。同时，系统会判断转换后的文字是不是一个有效问题（看有没有问号、疑问词等），并且30秒内相同的问题不会重复处理。

### 用户可以做什么？

| 用户故事 | 操作路径 | 完成状态 |
|---------|---------|----------|
| 开始录音 | 点击"开始面试"按钮 | ✅ |
| 停止录音 | 点击"停止面试"按钮 | ✅ |
| 实时看到识别结果 | 识别过程中显示临时文本 | ✅ |
| 自动过滤无效输入 | 非问题内容不会触发回答 | ✅ |
| 自动去重 | 30秒内重复问题不重复处理 | ✅ |

### 涉及几个页面？

| 页面名称 | 功能 | 状态 |
|---------|------|------|
| 面试页面 | 包含录音按钮和状态指示器 | ✅ |

### 关键数据结构

```
useSpeech (Composable)
├── isListening: boolean     # 是否正在监听
├── currentText: string      # 当前识别文本
├── startListening()         # 开始监听
└── stopListening()          # 停止监听

questionJudge (工具函数)
├── isValidQuestion(text)    # 判断是否为有效问题
├── containsQuestionWord()   # 是否包含疑问词
├── containsQuestionMark()   # 是否包含问号
└── isDuplicate(text)        # 是否为重复问题
```

### 关键架构决策

| ADR编号 | 决策内容 | 影响面 |
|---------|---------|--------|
| ADR-005 | 使用Web Speech API作为主方案 | 语音识别模块 |
| ADR-006 | 规则引擎优先，LLM兜底 | 问题判断逻辑 |

### 风险与注意事项

| 风险项 | 级别 | 说明 | 缓解措施 |
|--------|------|------|----------|
| 浏览器兼容性 | 🟡中 | Web Speech API在部分浏览器不支持 | 已添加兼容性检测，提示用户使用Chrome/Edge |
| 识别准确率 | 🟡中 | 免费API识别准确率有限 | 提供火山引擎ASR作为备用方案 |
