# 前端项目骨架模块 - 模块摘要

## 📌 一句话概括

面试虎的前端基础架构，提供Vue 3应用框架、路由管理、状态管理和UI基础组件，确保项目结构规范、可维护。

---

## 👤 面向产品/业务人员

### 这个模块做了什么？

前端项目骨架是面试虎Web应用的基础框架，就像一座房子的地基和主体结构。它搭建了Vue 3 + Vite + Tailwind CSS的技术栈，创建了首页、面试页面、配置弹窗等基础页面，配置了路由系统和状态管理，确保整个应用能够正常运行和扩展。

### 用户可以做什么？

| 用户故事 | 操作路径 | 完成状态 |
|---------|---------|----------|
| 打开应用看到首页 | 访问 http://localhost:5174 | ✅ |
| 点击开始面试进入面试页面 | 首页 → 点击"开始面试"按钮 | ✅ |
| 配置API Key和知识库 | 首页 → 点击设置图标 → 输入配置信息 → 保存 | ✅ |
| 浏览器兼容性检测 | 首页自动检测，不支持时显示提示 | ✅ |

### 涉及几个页面？

| 页面名称 | 功能 | 状态 |
|---------|------|------|
| 首页 | 应用入口，显示Logo和开始面试按钮 | ✅ |
| 面试页面 | 面试主界面，包含录音和对话展示 | ✅ |
| 配置弹窗 | 输入API Key和知识库ID等配置 | ✅ |

## 🔧 面向技术负责人

### 涉及接口清单

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | / | 首页 |
| GET | /interview | 面试页面 |

### 关键数据结构

```
InterviewStore (Pinia状态管理)
├── dialogues: Dialogue[]    # 对话列表
├── isRecording: boolean     # 是否正在录音
├── isGenerating: boolean    # 是否正在生成回答
├── currentQuestion: string  # 当前问题
└── config: Config           # 配置信息

Config
├── arkApiKey: string        # 火山引擎API Key
├── kbId: string             # 知识库ID
├── kbApiKey: string         # 知识库API Key
└── modelId: string          # 模型ID
```

### 关键架构决策

| ADR编号 | 决策内容 | 影响面 |
|---------|---------|--------|
| ADR-001 | 选择Vue 3 + Vite + Tailwind CSS技术栈 | 整个前端项目 |
| ADR-002 | 使用Pinia作为状态管理 | 状态共享组件 |
| ADR-003 | 使用Tailwind CSS原子化样式 | 所有UI组件 |

### 风险与注意事项

| 风险项 | 级别 | 说明 | 缓解措施 |
|--------|------|------|----------|
| 未处理404页面 | 🟡中 | 路由未匹配时无友好提示 | 建议后续添加404页面组件 |
| 无单元测试 | 🟡中 | 骨架阶段跳过测试 | 建议后续添加组件单元测试 |
