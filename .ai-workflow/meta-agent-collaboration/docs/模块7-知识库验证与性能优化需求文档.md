## 三层认知模型

### 第一层：事实锚定层（客户原文 + 编号）

**F1**：知识库不生效问题 — 接入了知识库，文件也上传了，但回答的内容不正确。问"你叫什么？"应该回答包含"刘朝相"，但实际回答了"我叫张伟"

**F2**：需要一个测试脚本 — 通过脚本测试知识库是否生效，而不是页面来回点、来回说，太麻烦

**F3**：加载状态问题 — "等待大模型接入..."这几个字不要，需要有个加载状态

**F4**：延迟问题 — 说完话后等了大概2秒才请求大模型，大模型返回信息也慢，需要优化响应速度

**F5**：期望测试标准 — 测试脚本问大模型"你叫什么？"，只有回复包含"刘朝相"才算正确

### 第二层：业务解读层（需求含义解读）

**B1**：知识库检索可能未正确命中 — 需要验证知识库检索环节是否返回了相关内容

**B2**：测试脚本需求 — 需要一个独立的命令行工具，能够：
- 直接调用知识库检索API，验证检索结果
- 调用大模型API（含知识库上下文），验证回答是否正确
- 提供明确的通过/失败判断标准

**B3**：加载状态优化 — 移除生硬的文字提示，替换为优雅的加载动画，提升用户体验

**B4**：延迟优化需求 — 需要从以下维度优化：
- 语音识别停顿检测时间（当前2秒）是否可缩短
- 知识库检索与大模型调用是否可并行
- 减少不必要的等待环节

**B5**：验证标准 — "你叫什么？" → 回答包含"刘朝相" = 知识库生效
