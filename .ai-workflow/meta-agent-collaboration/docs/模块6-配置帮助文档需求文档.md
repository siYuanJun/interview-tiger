## 三层认知模型

### 第一层：事实锚定层（客户原文 + 编号）

**F1**：新用户使用系统时，可能不知道如何配置火山引擎的大模型以及知识库的相应配置

**F2**：需要列出典型问题，包括但不限于：
- 火山大模型的模型名字该从哪获得？
- 大模型的 Token 如何获得？
- 接口的 API Key 如何获得？
- 知识库的名称如何获得？
- 知识库的 API Key 如何获得？

**F3**：在配置相关 Key 的弹窗中添加帮助功能（类似问号图标）

**F4**：帮助内容展示形式为：列出常见问题 + 简短描述 + 跳转文档链接

**F5**：参考文档链接：`https://my.feishu.cn/docx/AzijduiLDoSnsgxsaQsc7O3pnIb?from=from_copylink`

### 第二层：业务解读层（需求含义解读）

**B1**：提升新用户上手体验，降低配置门槛

**B2**：通过帮助入口解决用户在配置阶段的常见困惑

**B3**：采用「问题列表 + 跳转文档」的轻量形式，避免弹窗内容过长

**B4**：帮助入口应直观可见（问号图标），点击后展开常见问题列表

**B5**：每个问题项包含：问题标题、简短说明、点击跳转详细文档
