## 三层认知模型

### 第一层：事实锚定层（客户原文 + 编号）

**R01**：保留原始文件 — 上传到本地知识库的文件不要删，要持久化保存

**R02**：支持多文件上传 — 确认当前已支持，需在文档列表中体现

**R03**：上传后持久化，不是临时的 — 当前 `/tmp/` 用完即删（local_knowledge.py:170-172），需改为持久存储

**R04**：需要下载能力（隐含需求） — 既然保存了原文件，用户应该能在前端下载

### 第二层：业务解读层（需求含义解读）

**B1**：文件持久保存
- 在 `upload()` 方法中，当前 `finally: os.remove(temp_path)` 改为将文件复制到持久目录
- 持久目录建议：`./data/chroma/originals/`，与 ChromaDB 向量数据同属 `LOCAL_KB_DATA_DIR` 下
- 文件名策略：使用 `{doc_id}_{original_filename}` 避免冲突

**B2**：生命周期联动
- 删除文档时同步删除原始文件（`delete_doc()` 方法需追加文件删除逻辑）
- 清空知识库时同步清空所有原始文件（`clear()` 方法需追加目录清理逻辑）
- 上传新文件覆盖旧文件？否，文件名冲突用 doc_id 前缀隔离

**B3**：前端展示增强
- 文档列表 API 返回 `file_size` 字段（当前 metadata 已有，list_docs 未暴露）
- 前端 ConfigModal 文档列表中每项增加下载按钮
- 新增后端 API：`GET /api/local_kb/download/{doc_id}`

**B4**：Docker 持久化
- ChromeDB 数据已在 `./backend:/app` 卷挂载下，`originals/` 同理
- 如果以后拆独立卷挂载，`LOCAL_KB_DATA_DIR` 需要一致

## 详细设计方案

### 1. 配置变更

```python
# config.py 新增
LOCAL_KB_ORIGINALS_DIR = os.getenv("LOCAL_KB_ORIGINALS_DIR", 
    os.path.join(LOCAL_KB_DATA_DIR, "originals"))
```

环境变量：`LOCAL_KB_ORIGINALS_DIR=./data/chroma/originals`（默认值，可选覆盖）

### 2. upload() 改造

**当前代码（第170-172行）**：
```python
finally:
    if os.path.exists(temp_path):
        os.remove(temp_path)
```

**改造后**：
```python
finally:
    # 复制原始文件到持久目录
    originals_dir = Path(LOCAL_KB_ORIGINALS_DIR)
    originals_dir.mkdir(parents=True, exist_ok=True)
    doc_id = docs[0].metadata['doc_id'] if docs else uuid.uuid4().hex
    persistent_path = originals_dir / f"{doc_id}_{file.filename}"
    shutil.copy2(temp_path, persistent_path)
    # 清理临时文件
    if os.path.exists(temp_path):
        os.remove(temp_path)
```

### 3. delete_doc() 改造

删除向量后，追加删除原始文件逻辑：
```python
# 删除原始文件
originals_dir = Path(LOCAL_KB_ORIGINALS_DIR)
for f in originals_dir.glob(f"{doc_id}_*"):
    f.unlink()
```

### 4. clear() 改造

清空向量后，追加清空原始文件目录：
```python
# 清空原始文件
originals_dir = Path(LOCAL_KB_ORIGINALS_DIR)
if originals_dir.exists():
    shutil.rmtree(originals_dir)
    originals_dir.mkdir(parents=True, exist_ok=True)
```

### 5. 新增下载 API

```python
# local_kb.py 新增
@router.get("/local_kb/download/{doc_id}")
async def download_document(doc_id: str):
    """下载知识库原始文件"""
    originals_dir = Path(LOCAL_KB_ORIGINALS_DIR)
    matches = list(originals_dir.glob(f"{doc_id}_*"))
    if not matches:
        raise HTTPException(status_code=404, detail="原始文件不存在")
    
    file_path = matches[0]
    # 去掉 doc_id 前缀恢复原始文件名
    original_name = file_path.name[len(doc_id) + 1:]
    return FileResponse(file_path, filename=original_name)
```

### 6. list_docs() 增强

追加 `file_size_bytes` 字段（可读性用），前端展示文件体积。

### 7. 前端下载按钮

在 ConfigModal 文档列表中，每项增加下载图标按钮，调用 `/api/local_kb/download/{doc_id}` 下载。
