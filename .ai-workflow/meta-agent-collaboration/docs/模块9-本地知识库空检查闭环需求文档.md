## 三层认知模型

### 第一层：事实锚定层（客户原文 + 编号）

**R01**：默认使用本地知识库

**R02**：如果本地知识库没有配置文档，点击"开始面试"时需要提示

**R03**：直接弹出上传本地知识库文件的弹窗

**R04**：形成闭环

### 第二层：业务解读层（需求含义解读）

**B1**：产品默认推荐使用本地 ChromaDB 知识库，降低上手门槛（无需火山引擎账号）
- `DEFAULT_KB_PROVIDER` 从 `'volcengine'` 改为 `'local'`
- 新用户打开应用后，知识库类型默认选中"本地知识库"

**B2**：进入面试前做前置检查，如果本地知识库为空则阻断
- 调用 `GET /api/local_kb/stats` 检查 `total_chunks > 0`
- 空知识库无意义（无法做 RAG 检索），应引导用户先上传

**B3**：不阻断用户，而是直接打开上传弹窗
- 在 HomePage 点击"开始面试"→ 检测为空 → 自动打开 ConfigModal
- ConfigModal 预设为本地知识库 Tab，展示上传引导文案

**B4**：上传完成后无缝进入面试（闭环）
- ConfigModal 上传成功后 emit `uploaded` 事件
- HomePage 收到事件后关弹窗 + 自动导航到 `/interview`
- 用户感知："开始面试"→ 上传文档 → 直接进入面试

## 闭环流程图

```
用户点击"开始面试"
       ↓
  kb_provider === 'local'?
   ↓ 否                  ↓ 是
router→/interview    fetch /api/local_kb/stats
                          ↓
                    total_chunks > 0?
                ↓ 是              ↓ 否
          router→/interview   打开 ConfigModal
                              (pre-select local)
                              (标题："上传知识库文档")
                                   ↓
                            用户上传文档成功
                                   ↓
                            emit('uploaded')
                                   ↓
                            router→/interview
```
