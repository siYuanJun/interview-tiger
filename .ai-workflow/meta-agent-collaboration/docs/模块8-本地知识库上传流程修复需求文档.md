## 三、技术推导层（经机械臂验证）

### 3.1 完整调用链路追踪

```
用户操作                             前端                             后端
─────────                           ────                             ────
1. 打开设置弹窗 → ConfigModal opened
2. 切换到本地KB  → kbProvider = 'local'
3. 选择文件      → files = [file.pdf]
4. 点击"上传"    → uploadFiles()
                   ├─ POST /api/local_kb/upload          ─────────→  local_kb.py
                   ├─ FormData: files + chunk_size                 ├─ is_local_provider() ❌
                   └─ fetch API                                    └─ 返回 400: "当前未启用本地知识库"
5. 点击"保存配置" → saveConfig()
                   └─ localStorage.setItem('kb_provider','local')  （后端不知道）
```

### 3.2 根因定位

**Bug 位置**：`backend/app/routes/local_kb.py` — 全部 6 个路由的前置检查。

```python
# backend/app/routes/local_kb.py（所有路由）
if not is_local_provider():                                          # ← 问题所在
    raise HTTPException(status_code=400,
          detail="当前未启用本地知识库，请在设置中切换")
```

**`is_local_provider()` 的取值来源**：

```python
# backend/app/utils/kb_provider.py:29-30
def is_local_provider(provider_type: Optional[str] = None) -> bool:
    return get_kb_provider_type(provider_type) == "local"

# backend/app/utils/kb_provider.py:25-26
def get_kb_provider_type(provider_type: Optional[str] = None) -> str:
    return provider_type or KB_PROVIDER    # ← 路由没传参数，走到 KB_PROVIDER

# backend/config.py:13
KB_PROVIDER = os.getenv("KB_PROVIDER", "volcengine")  # ← 读服务端 .env
```

**关键事实**：`.env` 文件中 **未设置 KB_PROVIDER** 环境变量 → 默认值 `"volcengine"` → `is_local_provider()` 永久返回 `False` → 所有本地知识库 API 返回 400。

| 状态源 | 前端（localStorage） | 后端（.env） | 是否一致 |

### 3.4 为什么问答接口不受影响？

`/api/question/stream` 路由正确处理了动态 `kb_provider`：

```python
# backend/app/routes/question.py:16-33
class QuestionRequest(BaseModel):
    kb_provider: str = Field(default="", ...)  # ← 接收前端传入

def resolve_config(req: QuestionRequest):
    return {
        "kb_provider": req.kb_provider or KB_PROVIDER,  # ← 优先用请求参数
    }
```

但 `/api/local_kb/*` 路由中 **完全没有对应的请求参数接收逻辑**。

## 四、修复方案

### 方案：本地KB路由不再依赖服务端 KB_PROVIDER 环境变量

**思路**：去掉 `/api/local_kb/*` 中 `is_local_provider()` 的前置检查，改为无状态路由（调用方已明确选择了 local KB）。

**修改文件**：`backend/app/routes/local_kb.py`

**具体改动**：删除 6 个路由中的以下代码块（每处约 1 行）：

```python
if not is_local_provider():
    raise HTTPException(status_code=400, detail="当前未启用本地知识库，请在设置中切换")
```

**Backup**：同时删除文件顶部未使用的 `is_local_provider` import（保持代码整洁）。

**理由**：

1. **路由语义自洽**：路由前缀 `/api/local_kb/` 本身就声明了"这是本地知识库操作"，加一道 `is_local_provider()` 检查是多余的防御
2. **与问答路由一致**：`/api/question/*` 已正确支持 `kb_provider` 请求参数，本地KB路由不需要复读服务端配置来决定是否可以调用
3. **无安全风险**：本地KB操作（上传/删除/清空）操作的是本机 ChromaDB，没有访问外部服务的风险，不存在"用户绕过了配置限制"的安全问题
4. **修复简单**：仅删除 6 处 guard clause + 1 个 import， **零副作用**

### 补充优化（建议但非必须）

`config.py` 的 `KB_PROVIDER` 默认值建议改为 `"local"`，因为：
- 开源后新用户大概率先用本地知识库（免费）
- 火山引擎知识库需要额外申请 API Key

```python
# backend/config.py:13 建议
KB_PROVIDER = os.getenv("KB_PROVIDER", "local")
```
