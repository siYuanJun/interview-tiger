**1. Demo 简介**

* **是什么：** 一个 Web 端 AI 智能面试助手应用，基于浏览器实时语音识别，结合个人知识库，为求职者提供个性化的面试回答建议。

* **面向谁：** 正在求职、需要面试辅助的个人用户，以及希望提升面试表现、练习面试回答的求职者。

* **主要功能：**

  1. **实时语音识别**：自动识别面试官的提问，支持中文语音实时转文字，2秒停顿自动结束识别。
  
  ![首页截图](截图占位符-首页.png)

  2. **AI智能回答建议**：调用火山引擎方舟大模型，结合用户配置的知识库（需在火山引擎控制台提前上传个人简历/项目经历），生成贴合个人背景的结构化回答建议，支持STAR法则组织回答。

  ![面试中截图](截图占位符-面试中.png)

  3. **结构化对话展示**：左侧展示面试官问题，右侧展示 AI 生成的回答建议，支持展开/收起长回答，自动滚动到最新消息。

  ![对话详情截图](截图占位符-对话详情.png)

**2. Demo 创作思路**

* **灵感来源：** 身边很多朋友在面试时会紧张，明明准备好的内容却回答得不够理想。我想做一个工具，在面试过程中实时帮助用户，让他们更自信地应对面试官的提问。

* **想解决的问题：** 面试过程中用户容易紧张、回答不够结构化、无法充分展示个人优势。传统的面试准备方式是提前背诵，无法应对突发问题。

* **为什么做这个方向：** AI 技术的发展让实时辅助成为可能。通过语音识别+大模型+知识库的组合，可以实现个性化的面试辅助。相比通用的面试技巧网站，本产品结合用户个人背景，回答更具针对性。

**4. TRAE 实践过程**

整个开发过程完全基于 TRAE 完成，包括以下关键步骤：

**步骤一：需求分析与架构设计**

TRAE 帮助我梳理了产品需求，设计了完整的技术架构。前端使用 Vue 3 + Vite + Tailwind CSS，后端使用 Python + FastAPI + PostgreSQL。核心技术栈包括：浏览器 Web Speech API 语音识别、火山引擎方舟大模型（deepseek-v4-flash）、火山引擎企业知识库（RAG检索）。

**步骤二：前端界面开发**

TRAE 完成了所有前端组件的开发，包括：
- `HomePage.vue`：首页，包含开始面试按钮和配置入口，采用科技风格设计，使用 Lucide 图标和紫色主题
- `InterviewPage.vue`：面试页面，包含对话列表、录音状态指示器、控制按钮
- `DialogueItem.vue`：对话项组件，展示问题和回答，支持展开/收起功能
- `ConfigModal.vue`：配置弹窗，管理 API Key、知识库 ID 等

**步骤三：语音识别模块开发**

TRAE 实现了完整的语音识别逻辑，包括：
- `useSpeech.ts`：语音识别 Composable，处理麦克风权限、实时识别、2秒停顿自动结束、错误处理和自动重启机制
- 解决了语音识别重复启动的 bug（isStarting 锁 + generation 计数器 + isStopping 标志三重保护）

**步骤四：后端服务开发**

TRAE 完成了后端 API 服务，包括：
- `main.py`：FastAPI 入口，包含 CORS 配置、请求日志中间件、全局异常处理
- `llm.py`：火山引擎方舟大模型调用服务，支持流式/非流式两种模式
- `knowledge.py`：火山引擎知识库检索服务，使用 SignerV4 签名认证
- `prompt.py`：Prompt 拼接服务，支持知识库增强和无知识库降级方案
- `logger.py`：统一日志模块，支持短日志控制台输出和长日志文件存储（按天分割，保留30天）

**步骤五：部署脚本开发**

TRAE 编写了 `start.sh` 启动脚本，实现了：
- Docker 容器管理和 PostgreSQL 自动启动
- 端口冲突检测和自动清理
- 服务启动等待逻辑
