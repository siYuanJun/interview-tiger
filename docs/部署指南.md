# 面试虎 - 生产环境部署指南

## 概述

本文档详细介绍如何将面试虎项目部署到生产环境。项目采用前后端分离架构，需要分别部署前端、后端和数据库。

### 技术栈

| 组件 | 技术 | 部署平台 |
|------|------|----------|
| 前端 | Vue 3 + Vite + TypeScript | Vercel |
| 后端 | FastAPI + Uvicorn + Python | Render |
| 数据库 | PostgreSQL | Render |

### 部署架构

```
┌─────────────────┐     HTTP请求     ┌─────────────────┐     数据库连接     ┌─────────────────┐
│   Vercel        │ ────────────────> │   Render        │ ────────────────> │   Render        │
│   (前端)        │                   │   (后端API)     │                   │   (PostgreSQL)  │
│                 │ <───────────────  │                 │ <───────────────  │                 │
│  interview-     │    JSON响应       │  interview-     │    查询结果        │  interview-     │
│  tiger-frontend │                   │  tiger-backend  │                   │  tiger-db       │
└─────────────────┘                   └─────────────────┘                   └─────────────────┘
```

## 步骤一：部署 PostgreSQL 数据库（Render）

### 1.1 创建数据库

1. 登录 [Render Dashboard](https://dashboard.render.com)
2. 点击 **New** → **PostgreSQL**
3. 填写配置：
   - **Name**: `interview-tiger-db`
   - **Region**: 选择最近的区域（如 Oregon）
   - **Database Name**: `interview_tiger`
   - **User**: `postgres`
   - **Password**: 自动生成（记录下来）

4. 点击 **Create Database**

### 1.2 获取数据库连接字符串

创建完成后，在数据库详情页找到 **External Database URL**，格式如下：

```
postgresql://postgres:<password>@<host>:<port>/interview_tiger
```

**复制此 URL，后续配置后端时需要用到。**

## 步骤二：部署后端服务（Render）

### 2.1 创建 Web Service

1. 在 Render Dashboard 点击 **New** → **Web Service**
2. 选择 GitHub 仓库：`siYuanJun/interview-tiger-backend`
3. 点击 **Connect**

### 2.2 配置部署选项

| 配置项 | 值 |
|--------|-----|
| **Name** | `interview-tiger-backend` |
| **Region** | 与数据库相同的区域 |
| **Branch** | `main` |
| **Runtime** | `Python 3.11` |
| **Build Command** | `pip install -r requirements.txt` |
| **Start Command** | `uvicorn app.main:app --host 0.0.0.0 --port $PORT` |
| **Instance Type** | Free（免费版） |

### 2.3 配置环境变量

在 **Environment** 选项卡中添加以下环境变量：

| 变量名 | 值 |
|--------|-----|
| `ARK_API_KEY` | 您的方舟 API Key |
| `ARK_MODEL` | `deepseek-v4-flash-260425` |
| `ARK_BASE_URL` | `https://ark.cn-beijing.volces.com/api/v3` |
| `KB_API_KEY` | 您的知识库 API Key（sk-xxx 格式） |
| `KB_ID` | `siyuan_jianli` |
| `KB_PROJECT` | `default` |
| `KB_BASE_URL` | `https://api-knowledgebase.mlp.cn-beijing.volces.com` |
| `ASR_APP_ID` | 您的语音识别 App ID |
| `ASR_TOKEN` | 您的语音识别 Token |
| `ASR_CLUSTER` | `volcengine_streaming_common` |
| `ASR_WS_URL` | `wss://openspeech.bytedance.com/api/v2/asr` |
| `DATABASE_URL` | 步骤一获取的 PostgreSQL 连接字符串 |
| `ALLOWED_ORIGINS` | `https://interview-tiger-frontend.vercel.app` |
| `APP_DEBUG` | `false` |

### 2.4 部署

点击 **Create Web Service** 开始部署。

部署完成后，会获得后端 URL，格式如下：

```
https://interview-tiger-backend.onrender.com
```

### 2.5 验证后端

访问以下 URL 验证后端是否正常运行：

```
https://interview-tiger-backend.onrender.com/api/health
```

预期响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "healthy",
    "service": "面试虎 API",
    "version": "1.0.0"
  }
}
```

## 步骤三：部署前端（Vercel）

### 3.1 创建项目

1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击 **Add New** → **Project**
3. 选择 GitHub 仓库：`siYuanJun/interview-tiger-frontend`
4. 点击 **Import**

### 3.2 配置部署选项

| 配置项 | 值 |
|--------|-----|
| **Framework Preset** | `Vue.js` |
| **Build Command** | `npm run build` |
| **Output Directory** | `dist` |
| **Root Directory** | 留空（仓库根目录即为前端代码） |

### 3.3 配置环境变量

在 **Environment Variables** 中添加：

| 变量名 | 值 |
|--------|-----|
| `VITE_API_BASE_URL` | `https://interview-tiger-backend.onrender.com/api` |

### 3.4 部署

点击 **Deploy** 开始部署。

部署完成后，会获得前端 URL，格式如下：

```
https://interview-tiger-frontend.vercel.app
```

### 3.5 验证前端

1. 访问前端 URL，确认页面正常加载
2. 点击 **健康检查** 按钮，确认 API 连接正常
3. 配置 API Key 后测试完整功能

## 步骤四：配置 CORS

确保后端的 `ALLOWED_ORIGINS` 环境变量包含前端的实际域名：

```
https://interview-tiger-frontend.vercel.app
```

如果需要支持多个域名，使用逗号分隔：

```
https://interview-tiger-frontend.vercel.app,http://localhost:5173
```

## 常见问题

### Q1: 后端部署后无法连接数据库？

**检查项**：
1. 确认 `DATABASE_URL` 环境变量是否正确
2. 确认数据库和后端服务在同一区域
3. 确认数据库的 **External Database URL** 是否正确复制

### Q2: 前端无法调用后端 API？

**检查项**：
1. 确认前端 `VITE_API_BASE_URL` 是否指向正确的后端 URL
2. 确认后端 `ALLOWED_ORIGINS` 是否包含前端域名
3. 检查浏览器控制台是否有 CORS 错误

### Q3: Render 后端服务首次请求很慢？

**原因**：Render 免费版服务在空闲 15 分钟后会休眠，首次请求需要重新启动。

**解决方案**：
- 升级到付费版（$7/month），服务会持续运行
- 使用第三方服务（如 UptimeRobot）定期 ping 后端保持活跃

### Q4: 后端日志如何查看？

在 Render Dashboard 中进入后端服务，点击 **Logs** 标签查看实时日志。

### Q5: 前端构建失败？

**检查项**：
1. 确认 `package.json` 中 `build` 脚本存在
2. 确认所有依赖已正确安装
3. 检查 Vercel 构建日志中的具体错误信息

## 更新部署

### 更新前端

1. 在本地修改前端代码
2. 提交并推送到 GitHub：
   ```bash
   cd frontend
   git add .
   git commit -m "更新描述"
   git push origin main
   ```
3. Vercel 会自动触发部署

### 更新后端

1. 在本地修改后端代码
2. 提交并推送到 GitHub：
   ```bash
   cd backend
   git add .
   git commit -m "更新描述"
   git push origin main
   ```
3. Render 会自动触发部署
