# 面试虎 - 后端接口全貌

## 1. HTTP 接口总览

### 健康检查

端点数量: 1

| 请求方法 | 完整路径 | 参数 | 返回值类型 | 接口说明 |
|----------|----------|------|------------|----------|
| GET | /api/health | 无 | JSON | 服务健康检查，返回服务运行状态和版本信息，用于 Docker 健康检查或前端连接测试 |

### 配置管理

端点数量: 2

| 请求方法 | 完整路径 | 参数 | 返回值类型 | 接口说明 |
|----------|----------|------|------------|----------|
| POST | /api/config | ark_api_key(str), kb_id(str), model_id(str) | JSON | 保存配置并验证有效性，接收前端传来的 API Key、知识库 ID 和模型 ID，后端仅做转发验证 |
| GET | /api/config | 无 | JSON | 获取当前配置状态（返回脱敏信息），仅返回 API Key 是否已配置的布尔值，不返回实际密钥 |

### 知识库检索

端点数量: 1

| 请求方法 | 完整路径 | 参数 | 返回值类型 | 接口说明 |
|----------|----------|------|------------|----------|
| POST | /api/search | query(str), kb_id(str), kb_api_key(str), top_k(int) | JSON | 检索知识库，后端代理调用火山引擎 RAG 接口，返回相关文档片段，支持前端传入配置覆盖默认值 |

### 大模型生成

端点数量: 2

| 请求方法 | 完整路径 | 参数 | 返回值类型 | 接口说明 |
|----------|----------|------|------------|----------|
| POST | /api/generate | prompt(str), ark_api_key(str), model_id(str), stream(bool), temperature(float), max_tokens(int) | JSON | 大模型生成回答（非流式），接收完整 Prompt 文本，调用火山引擎方舟大模型返回完整回答 |
| POST | /api/generate/stream | prompt(str), ark_api_key(str), model_id(str), stream(bool), temperature(float), max_tokens(int) | SSE | 大模型生成回答（流式 SSE），实时推送生成的文本片段，适用于长文本生成场景 |

### 问题处理（核心 API）

端点数量: 2

| 请求方法 | 完整路径 | 参数 | 返回值类型 | 接口说明 |
|----------|----------|------|------------|----------|
| POST | /api/question | question(str), ark_api_key(str), model_id(str), kb_id(str), kb_api_key(str), stream(bool) | JSON | 处理面试问题（非流式），集成知识库检索 + 大模型回答，知识库无结果时自动降级为联网搜索 |
| POST | /api/question/stream | question(str), ark_api_key(str), model_id(str), kb_id(str), kb_api_key(str), stream(bool) | SSE | 处理面试问题（流式 SSE），实时推送回答片段，支持状态更新和错误处理 |

### 语音识别

端点数量: 1

| 请求方法 | 完整路径 | 参数 | 返回值类型 | 接口说明 |
|----------|----------|------|------------|----------|
| WEBSOCKET | /api/asr/stream | ws(WebSocket) | JSON | 语音识别流式接口，前端通过 WebSocket 发送 PCM 音频数据，后端代理到火山引擎 ASR 实时返回识别文本 |

### 本地知识库

端点数量: 7

| 请求方法 | 完整路径 | 参数 | 返回值类型 | 接口说明 |
|----------|----------|------|------------|----------|
| POST | /api/local_kb/upload | files(List[UploadFile]), chunk_size(int), chunk_overlap(int) | JSON | 上传文件到本地知识库，支持 md/txt/pdf/docx/json，自动切片存入 ChromaDB 向量库，并持久化原始文件 |
| GET | /api/local_kb/list | 无 | JSON | 获取已上传文档列表，返回文档名、doc_id、切片数、文件大小 |
| POST | /api/local_kb/search | query(str), top_k(int) | JSON | 检索本地知识库，返回相关文档片段及相似度分数 |
| GET | /api/local_kb/stats | 无 | JSON | 获取知识库统计信息，包含总切片数、数据目录、Embedding 模型名 |
| GET | /api/local_kb/download/{doc_id} | doc_id(str) | File | 下载知识库原始文件，返回原始上传文件 |
| DELETE | /api/local_kb/delete/{doc_id} | doc_id(str) | JSON | 删除指定文档的所有切片及原始文件 |
| DELETE | /api/local_kb/clear | 无 | JSON | 清空整个本地知识库，删除所有切片和原始文件 |

### 对话管理

端点数量: 4

| 请求方法 | 完整路径 | 参数 | 返回值类型 | 接口说明 |
|----------|----------|------|------------|----------|
| POST | /api/transcript | text(str), session_id(Optional[str]) | JSON | 接收前端识别的文本，进行问题有效性判断，有效问题存入数据库 |
| GET | /api/dialogues | session_id(Optional[str]) | JSON | 获取当前会话的所有对话记录，支持按 session_id 筛选 |
| DELETE | /api/dialogues | session_id(Optional[str]) | JSON | 清空对话记录，支持按 session_id 筛选删除 |
| PUT | /api/dialogues/{dialogue_id} | dialogue_id(str), answer(Optional[str]) | JSON | 更新对话记录，主要用于更新大模型生成的回答内容 |

**接口总数**: 20

## 2. 接口详细说明

### 2.1 健康检查

#### GET /api/health

**响应示例**:
```json
{
    "code": 0,
    "message": "ok",
    "data": {
        "status": "healthy",
        "service": "面试虎 API",
        "version": "1.0.0"
    }
}
```

### 2.2 配置管理

#### POST /api/config

**请求体**:
| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| ark_api_key | string | 是 | 火山引擎方舟 API Key | - |
| kb_id | string | 是 | 知识库 ID | - |
| model_id | string | 否 | 模型 ID | deepseek-v4-flash-260425 |

**响应示例**:
```json
{
    "code": 0,
    "message": "配置已保存",
    "data": {
        "ark_api_key_configured": true,
        "kb_id": "siyuan_jianli",
        "model_id": "deepseek-v4-flash-260425"
    }
}
```

#### GET /api/config

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "ark_api_key_configured": true,
        "kb_id": "siyuan_jianli",
        "model_id": "deepseek-v4-flash-260425"
    }
}
```

### 2.3 知识库检索

#### POST /api/search

**请求体**:
| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| query | string | 是 | 检索查询文本 | - |
| kb_id | string | 否 | 知识库 ID（留空使用 .env 配置） | - |
| kb_api_key | string | 否 | 知识库 API Key（留空使用 .env 配置） | - |
| top_k | int | 否 | 返回结果数量 | 5 |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "chunks": [
            {"content": "知识库文档内容...", "score": 0.85, "doc_name": "知识库文档"}
        ]
    }
}
```

### 2.4 大模型生成

#### POST /api/generate

**请求体**:
| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| prompt | string | 是 | 完整 Prompt 文本 | - |
| ark_api_key | string | 否 | 火山引擎 API Key | - |
| model_id | string | 否 | 模型 ID | deepseek-v4-flash-260425 |
| stream | bool | 否 | 是否流式输出 | false |
| temperature | float | 否 | 温度参数 | 0.7 |
| max_tokens | int | 否 | 最大输出 token 数 | 1000 |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {"answer": "大模型生成的回答..."}
}
```

#### POST /api/generate/stream

**请求体**: 同上

**响应类型**: SSE（Server-Sent Events）

**响应格式**:
```
data: {"chunk": "第一段文本"}

data: {"chunk": "第二段文本"}

data: [DONE]
```

### 2.5 问题处理

#### POST /api/question

**请求体**:
| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| question | string | 是 | 面试官问题文本 | - |
| ark_api_key | string | 否 | 火山引擎 API Key | - |
| model_id | string | 否 | 模型 ID | deepseek-v4-flash-260425 |
| kb_id | string | 否 | 知识库 ID | - |
| kb_api_key | string | 否 | 知识库 API Key | - |
| stream | bool | 否 | 是否流式输出 | true |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "answer": "大模型生成的回答...",
        "knowledge_used": true,
        "web_search_used": false,
        "source_chunks": []
    }
}
```

#### POST /api/question/stream

**请求体**: 同上

**响应类型**: SSE（Server-Sent Events）

**响应格式**:
```
data: {"type": "status", "message": "正在生成回答..."}

data: {"type": "chunk", "content": "第一段文本"}

data: {"type": "chunk", "content": "第二段文本"}

data: [DONE]
```

### 2.6 语音识别

#### WEBSOCKET /api/asr/stream

**前端 → 后端消息格式**:
```json
{"type": "audio", "data": "<base64编码的PCM>"}
{"type": "finish"}
{"type": "config", "app_id": "...", "token": "..."}
```

**后端 → 前端消息格式**:
```json
{"type": "result", "text": "识别文本", "is_final": false}
{"type": "error", "message": "错误信息"}
{"type": "status", "message": "状态信息"}
```

### 2.7 本地知识库

#### POST /api/local_kb/upload

**请求方式**: `multipart/form-data`

| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| files | File[] | 是 | 上传的文件列表，支持 .md/.txt/.pdf/.docx/.json | - |
| chunk_size | int | 否 | 文本切片大小（字符数） | 500 |
| chunk_overlap | int | 否 | 切片重叠大小（字符数） | 50 |

**响应示例**:
```json
{
    "code": 0,
    "message": "成功上传 1 个文件",
    "data": [
        {
            "filename": "01-自我介绍与核心必答题.md",
            "doc_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
            "chunks": 25
        }
    ]
}
```

#### GET /api/local_kb/list

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": [
        {
            "doc_name": "01-自我介绍与核心必答题.md",
            "doc_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
            "chunks": 25,
            "file_size_bytes": 10406
        }
    ]
}
```

#### POST /api/local_kb/search

**请求体**:
| 字段 | 类型 | 必填 | 说明 | 默认值 |
|------|------|------|------|--------|
| query | string | 是 | 检索查询文本，长度 1-500 | - |
| top_k | int | 否 | 返回结果数量，范围 1-10 | 5 |

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "chunks": [
            {
                "content": "大家好，我是张三，有5年Java开发经验...",
                "score": 0.8521,
                "doc_name": "01-自我介绍与核心必答题.md",
                "doc_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
            }
        ]
    }
}
```

#### GET /api/local_kb/stats

**响应示例**:
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "total_chunks": 25,
        "data_dir": "./data/chroma",
        "embedding_model": "bge-large-zh-v1.5"

}
```

#### GET /api/local_kb/download/{doc_id}

**路径参数**:

|------|------|------|------|
| doc_id | string | 是 | 文档唯一 ID（上传时返回的 doc_id） |

**响应**: 二进制文件流，Content-Type 根据原始文件类型自动设置。

**路径参数**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| doc_id | string | 是 | 文档唯一 ID |

**响应示例**:
```json

        "doc_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
```

**响应示例**:
```json

    "code": 0,

        "chunks_deleted": 25
    }
```

## 3. 服务间调用

| 调用方 | 被调用方 | 方法/路径 | 用途 | 认证方式 |
|--------|----------|-----------|------|----------|
| backend | 火山引擎方舟 | POST /api/v3/chat/completions | 大模型调用 | API Key（Bearer Token） |
| backend | 火山引擎知识库 | POST /api/v1/rag/retrieve | 知识库检索 | API Key（Bearer Token） |
| backend | 火山引擎 ASR | WebSocket | 语音识别 | App ID + Token |

## 4. 消息队列通信

## 5. 通用响应格式

所有 API 接口统一返回以下格式：

```json
{
    "code": 0,
    "message": "success",
    "data": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，0 表示成功，非 0 表示错误 |
| message | string | 状态消息 |
| data | any | 业务数据，成功时返回具体内容，失败时可能为 null |
