## RAG 3.0 升级实施（meta-feature-dev 工作流）

### 背景
- 使用 meta-feature-dev 7 阶段工作流，对 interview-tiger 实施 RAG 从 2.5 → 3.0 升级（路径A）
- 完成 Stages 1-5（规划阶段），用户批准后进入 Stage 6（执行）

### 新增文件
- `backend/app/services/hybrid_retriever.py` — BM25+Dense 混合检索，RRF 融合，EnsembleRetriever
- `backend/app/services/query_rewriter.py` — 15 个面试领域同义词映射，最多生成 3 个查询变体
- `backend/app/services/memory.py` — 会话级内存缓存，TTL 过期 + 最大会话数淘汰，追问模式检测
- `backend/app/services/validator.py` — LLM 批量内容校验，3 秒超时兜底，YES/NO 解析

### 修改文件
- `backend/app/services/local_knowledge.py` — 集成 4 个 RAG3 模块，懒加载初始化，全管道搜素
- `backend/config.py` — 新增 8 个 RAG3 配置变量
- `backend/app/routes/question.py` — 新增 session_id 字段支持
- `backend/requirements.txt` — 添加 jieba==0.42.1

### Docker 测试问题
- 在 requirements.txt 添加 jieba 导致 Docker 层缓存失效，需要重新下载 ~800MB PyTorch
- 尝试预下载 wheels 到 backend/wheels/ — 不兼容（macOS wheels 无法在 Linux Docker 使用）
- **最终方案**：不重建镜像，直接 `docker exec pip install jieba`，避免重下所有依赖
- 已删除 backend/wheels/ 目录
